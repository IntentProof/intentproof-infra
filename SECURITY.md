# Security Policy

## Reporting A Vulnerability

Report suspected vulnerabilities to `security@intentproof.io`. If you need to
share sensitive details before a public encryption key is published, open a
private GitHub Security Advisory draft in the affected repository or send an
initial unclassified message to arrange a secure channel.

Please include:

- Affected repository, package, binary, image, or hosted endpoint.
- Impact and exploitation conditions.
- Reproduction steps or proof-of-concept details when safe to share.
- Whether the issue is already public or shared with anyone else.

## Response Process

IntentProof follows the coordinated security-release process in
[`SECURITY-PROCESS.md`](SECURITY-PROCESS.md). That process defines severity
tiers, SLAs, embargo handling, patch release steps, customer notification, and
post-mortem requirements.

Do not report security vulnerabilities through public GitHub Issues unless the
issue is already public and contains no sensitive exploitation detail.
